## [Using Git's --author Flag to Block AI Bot Spam on GitHub](https://archestra.ai/blog/only-responsible-ai) ⭐️ 7.0/10

The author of Archestra describes a method to filter out AI-generated spam pull requests on GitHub by using Git's --author flag to check whether the commit author matches a known AI bot pattern, and then automatically closing such PRs. As AI-generated code contributions become common, open-source maintainers face an influx of low-quality or spammy PRs that waste maintainer time. This approach provides a simple, automated defense that can be adopted by other repositories to maintain project quality. The technique relies on the fact that many AI bots leave identifiable commit author names (e.g., 'github-actions[bot]') that can be captured by a regex pattern. However, this approach is not foolproof, as bots can easily modify their author information.

hackernews · ildari · May 18, 15:24 · [Discussion](https://news.ycombinator.com/item?id=48181125)

**Background**: Git's --author flag allows filtering commits by the author's name or email. GitHub pull requests often contain commits from automated accounts, including AI bots that submit unsolicited changes. The open-source community is increasingly concerned about such spam, which burdens maintainers with reviewing low-quality contributions.

**Discussion**: Commenters highlighted security implications: contributors with merged PRs gain elevated privileges, which malicious bots could exploit. Others expressed frustration with GitHub's lack of basic spam prevention, and suggested systems like ELO ratings to filter contributions based on quality rather than origin.
