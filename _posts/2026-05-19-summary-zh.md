## [利用 Git 的 --author 参数阻挡 GitHub 上的 AI 机器人垃圾信息](https://archestra.ai/blog/only-responsible-ai) ⭐️ 7.0/10

Archestra 的作者描述了一种方法，通过使用 Git 的 --author 参数检查提交作者是否匹配已知的 AI 机器人模式，并自动关闭此类 PR，从而过滤 GitHub 上的 AI 生成垃圾拉取请求。 随着 AI 生成的代码贡献变得普遍，开源维护者面临大量低质量或垃圾 PR 的涌入，浪费维护者时间。此方法提供了一种简单的自动化防御，其他仓库可以借鉴以维护项目质量。 该技术依赖于许多 AI 机器人会留下可识别的提交作者名称（例如 'github-actions[bot]'），可以通过正则表达式捕获。但这种方法并非万无一失，因为机器人可以轻易修改作者信息。

hackernews · ildari · May 18, 15:24 · [社区讨论](https://news.ycombinator.com/item?id=48181125)

**背景**: Git 的 --author 参数允许按作者名称或邮箱过滤提交。GitHub 拉取请求常包含来自自动化账户的提交，包括提交非请求变更的 AI 机器人。开源社区日益关注此类垃圾信息，它们给维护者带来了审查低质量贡献的负担。

**社区讨论**: 评论者强调了安全隐患：已合并 PR 的贡献者会获得更高权限，恶意机器人可能利用这一点。其他人对 GitHub 缺乏基本垃圾信息预防表示不满，并建议引入像 ELO 评级这样的系统，根据质量而非来源过滤贡献。
